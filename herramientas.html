// General (Linux/Kali) //

Error: 'Permission denied' (Permiso denegado)

El error más común. No tienes privilegios para ejecutar la acción (ej. abrir un puerto, leer un archivo protegido, iniciar una captura de red).

Solución 1 (Root): Ejecuta el comando con privilegios de superusuario.
sudo !! (!! repite el último comando)

Solución 2 (Permisos): Si es un script, dale permisos de ejecución.
chmod +x mi_script.sh

Error: 'command not found' (Comando no encontrado)

El sistema no encuentra el ejecutable de la herramienta que intentas usar.

Solución 1 (Instalación): Asegúrate de que la herramienta esté instalada.
sudo apt install [nombre_herramienta]

Solución 2 (PATH): Si está instalada, asegúrate de que la carpeta que la contiene esté en tu variable $PATH del sistema.
echo $PATH

Error: 'File or directory not found' (Archivo no encontrado)

Estás intentando acceder a un archivo (wordlist, script, config) que no está en la ruta que especificaste.

Solución: Verifica la ruta. Usa pwd (imprimir directorio actual) y ls -l (listar archivos) para confirmar que estás donde crees estar y que el archivo existe.

// Nmap //

Error: 'Host seems down' (El host parece caído)

Nmap cree que el objetivo no está en línea porque su "ping" inicial (ICMP) fue bloqueado, usualmente por un firewall.

Solución: Forza el escaneo asumiendo que está vivo. Usa el flag -Pn.
sudo nmap -Pn -sV target.com

Error: 'Note: Host is listening on... but not responding to TCP...'

Nmap sabe que el puerto está abierto (ej. por un escaneo SYN) pero el servicio en ese puerto te está ignorando. Esto es común en puertos filtrados por firewall.

Solución: El puerto está filtered. Intenta usar diferentes tipos de escaneo (-sT, -sF, -sX) o cambia tu dirección IP de origen.

// Metasploit (msfconsole) //

Error: 'Exploit failed: The connection was refused...'

El exploit se ejecutó, pero el objetivo rechazó activamente la conexión. Esto puede significar:

1. El puerto (RPORT) está cerrado en el objetivo.
2. El servicio no está corriendo en ese puerto.
3. Un firewall bloqueó la conexión.

Solución: Confirma con Nmap que el puerto está open y que el servicio (SRVHOST, RHOST) es correcto.

Error: 'No payload configured, defaulting to...'

Estás usando un exploit sin especificar *qué* quieres que haga después (ej. windows/meterpreter/reverse_tcp). Metasploit usará uno por defecto, pero rara vez es el que necesitas.

Solución: Usa el comando set payload [nombre_payload] después de seleccionar tu exploit.

Error: 'Session 1 died' (La sesión murió)

Obtuviste una sesión (Meterpreter, shell) pero se cerró inmediatamente. Esto suele pasar si el exploit es inestable, el antivirus lo detectó y lo mató, o el payload no es compatible con el sistema.

Solución: Intenta un payload diferente (ej. stageless vs staged) o migra a un proceso más estable (migrate [PID]) tan pronto como entres.

// Wireshark //

Error: 'No interfaces found' (No se encontraron interfaces)

Wireshark no puede encontrar ninguna tarjeta de red (WiFi, Ethernet) para capturar paquetes.

Solución (Linux): Probablemente lo iniciaste sin permisos. Cierra Wireshark y ejecútalo como root: sudo wireshark. O (mejor) añade tu usuario al grupo wireshark.

Tráfico HTTPS aparece como 'TCP' o 'TLSv1.2' y no se puede leer.

Esto no es un error, es encriptación. Wireshark no puede leer tráfico HTTPS por defecto.

Solución: Si controlas el cliente (ej. tu propio navegador), puedes exportar las "Pre-Master-Secret keys" desde el navegador e importarlas en Wireshark (Edit > Preferences > Protocols > SSL > (Pre)-Master-Secret log filename). Es un proceso avanzado.

// SQLMap //

Error: 'all tested parameters do not appear to be injectable'

SQLMap probó los parámetros (ej. ?id=) y no pudo confirmar una inyección SQL con sus pruebas básicas.

Solución: Aumenta la intensidad de las pruebas. Usa --level=3 (añade más payloads) y --risk=3 (añade pruebas más peligrosas). También puedes estar siendo bloqueado por un WAF (Web Application Firewall).

Error: 'target URL is not stable' (El sitio es inestable)

SQLMap compara la página normal con la página "atacada". Si la página cambia mucho en cada recarga (ej. por anuncios dinámicos), SQLMap se confunde.

Solución: Intenta encontrar una página más estable o usa la opción --string="[TEXTO_FIJO]" para decirle a SQLMap qué texto siempre debe estar presente en la página válida.

// Aircrack-ng //

Error: 'Fixed channel [interface]: -1'

Intentaste usar airodump-ng o aireplay-ng en una interfaz (ej. wlan0) que no está en "Modo Monitor".

Solución: Debes poner la tarjeta en modo monitor primero.
sudo airmon-ng start wlan0 (Esto creará una nueva interfaz, ej. wlan0mon)
sudo airodump-ng wlan0mon

Error: 'Got 0 IVs' (Tengo 0 IVs) - (Ataque WEP)

Estás intentando crackear una red WEP, pero airodump-ng no está capturando "paquetes de datos" (IVs), solo beacons. Esto pasa si no hay nadie conectado o usándola.

Solución: Necesitas forzar tráfico. Usa aireplay-ng --arpreplay -b [BSSID] -h [CLIENT_MAC] wlan0mon para re-inyectar paquetes ARP y generar tráfico.

// Hashcat //

Error: 'No hashes loaded' (No se cargaron hashes)

El programa no reconoce el formato de tu hash o el archivo está mal formateado.

Solución: Asegúrate de que el archivo de hash contenga *solo* el hash (ej. 5f4dcc3b5aa765d61d8327deb882cf99) y especifica el modo correcto (-m [CODE]). Usa la web de ejemplos de Hashcat para encontrar el código de tu hash (ej. -m 0 para MD5).

Error: 'Device #1: Kernel... insufficient memory' (Memoria insuficiente)

Tu tarjeta gráfica (GPU) no tiene suficiente VRAM para manejar el ataque que estás pidiendo (común con hashes "lentos" como bcrypt o en ataques de diccionario muy grandes).

Solución: Cierra otros programas que usen la GPU (juegos, videos). Si eso no funciona, reduce el tamaño del "workload" con -w 2 o -w 1, aunque esto reducirá la velocidad.

// Burp Suite //

Error: 'SSLHandshakeException' (Error de certificado)

Tu navegador no confía en el certificado "falso" que Burp usa para interceptar tráfico HTTPS.

Solución: Con Burp abierto y configurado como proxy, ve en tu navegador a http://burp/ (o http://127.0.0.1:8080). Descarga el "CA Certificate" e impórtalo en la sección de "Autoridades Confiables" en la configuración de certificados de tu navegador.
